# users

Приложение `users` отвечает за пользователей и аутентификацию.

Сейчас реализовано:
- кастомная модель `User` (роль: student/parent/teacher/methodist);
- JWT-аутентификация (SimpleJWT);
- API: регистрация/логин/получить текущего пользователя.

## Модели

### User
Расширение `AbstractUser` с дополнительным полем `role`.

Зачем:
- единая точка управления ролями и доступом к сценариям;
- в будущем — разная логика для ученика/родителя/учителя/методиста.

Пример:
- `User(username="ivan", role="student")`
