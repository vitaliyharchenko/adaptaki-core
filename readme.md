# Adaptaki

## Суть проекта

Adaptaki — это образовательная платформа с **графом знаний** и **адаптивной логикой обучения**.

Ключевая идея:

> система не просто выдаёт задания,  
> а диагностирует уровень ученика, понимает пробелы и строит индивидуальную траекторию обучения.

Платформа ориентирована на школьников и их родителей, а также на преподавателей, работающих с группами.

---

## Архитектурная идея

Adaptaki строится по принципу **backend-first**.

-   Backend — ядро системы и единственный источник истины
-   Frontend — тонкий, заменяемый клиент
-   Бизнес-логика не живёт в UI
-   Все клиенты работают через API

Платформа изначально проектируется как **мульти-клиентская**:

-   Web (React SPA)
-   Mobile (в будущем)
-   Telegram Bot / Mini App (в будущем)

---

## Текущий статус проекта

Проект находится на стадии **формирования инженерного фундамента**.

### Уже реализовано

-   Backend на Django + DRF
-   Кастомная модель пользователя
-   JWT-аутентификация
-   CORS для локальной разработки
-   React SPA (Vite + TypeScript)
-   End-to-end авторизация (React ↔ Django)

### Сознательно не реализовано (пока)

-   Доменные модели обучения
-   Граф знаний
-   Генерация заданий
-   Интеграция ИИ
-   Telegram-клиенты

Это будет добавляться **после стабилизации основы**.

## Архитектурные принципы (важно)

Эти принципы — **договорённость проекта**:

1. Backend содержит всю бизнес-логику
2. Frontend не принимает решений за систему
3. JWT — единственный способ авторизации клиентов
4. Архитектура важнее скорости реализации фич
5. Проект развивается через вертикальные срезы, а не через фич-листы

---

## Как работать с проектом

### Backend

-   Django используется как API и ORM
-   Бизнес-логика выносится из views
-   Новые сценарии добавляются как отдельные use cases

### Frontend

-   React SPA — тонкий клиент
-   Вся логика обучения остаётся на backend
-   UI может быть переписан без изменения API

## Для ИИ-ассистентов (Cursor, ChatGPT)

Этот репозиторий:

-   использует Django как backend API
-   использует React SPA как frontend
-   придерживается clean-architecture-подхода без фанатизма
-   находится на ранней стадии, где важнее структура, чем фичи

При продолжении разработки:

-   сохраняй текущие архитектурные принципы
-   не усложняй преждевременно
-   развивай систему через понятные сценарии
